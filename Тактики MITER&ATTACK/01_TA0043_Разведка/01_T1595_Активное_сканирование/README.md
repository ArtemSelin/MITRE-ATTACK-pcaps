# Техника: Активное сканирование (Active Scanning — T1595)

**Активное сканирование** — это метод разведки, при котором нарушитель напрямую взаимодействует с целевыми системами в сети, отправляя им специально сформированные пакеты. Цель — определить структуру сети, обнаружить доступные сервисы, операционные системы и найти уязвимости для последующей эксплуатации.

В этом разделе собраны дампы трафика, полученные при работе популярных инструментов сканирования и анализа защищенности.

---

## 📂 Описание подтехник и состав дампов

Внутри этого каталога файлы распределяются по следующим подтехникам:

### 1. T1595.001 — Сканирование IP-адресов (IP Addresses Scanning)
*   **Суть:** Массовый поиск «живых» хостов в заданных сетевых диапазонах (Host Discovery / Ping Sweeps).
*   **Сетевой след (что искать в PCAP):** 
    *   Шквал ICMP Echo Request пакетов к последовательным IP-адресам.
    *   Массовые ARP-запросы (если сканирование идет внутри одного L2-сегмента).
    *   TCP SYN-пакеты, отправляемые на один или несколько стандартных портов (например, 80, 443, 22) по всей сети для быстрого обнаружения серверов.
*   **Примеры утилит:** `nmap -sn`, `masscan`, `fping`.

### 2. T1595.002 — Сканирование уязвимостей (Vulnerability Scanning)
*   **Суть:** Направленный поиск известных уязвимостей (CVE), незакрытых портов и ошибок конфигурации на обнаруженных хостах.
*   **Сетевой след (что искать в PCAP):**
    *   **Сетевой уровень:** Классические типы сканирования портов: *TCP SYN Stealth* (полуоткрытое), *TCP Connect* (полное установление сессии), скрытые режимы (*FIN, NULL, XMAS*).
    *   **Прикладной уровень (L7):** Интенсивный обмен данными, отправка специфичных запросов (payloads) под конкретные CVE, проверка баннеров служб (Banner Grabbing). Наличие сигнатурных строк сканеров в заголовках пакетов.
*   **Примеры утилит:** `nmap -sS / -sC / --script`, `Nessus`, `OpenVAS`, `Nikto`.

### 3. T1595.003 — Сканирование по словарю (Wordlist Scanning)
*   **Суть:** Перебор путей, директорий, файлов и параметров на веб-серверах с использованием готовых списков (словарей). Применяется для поиска скрытых панелей администратора, забытых бэкапов (`.bak`, `.zip`) или конфигурационных файлов.
*   **Сетевой след (что искать в PCAP):**
    *   Огромный поток последовательных HTTP GET/HEAD запросов к одному веб-серверу за минимальный промежуток времени.
    *   Большое количество ответов сервера со статус-кодами `404 Not Found` или `403 Forbidden` и редкие ответы `200 OK`.
*   **Примеры утилит:** `Gobuster`, `Dirb`, `Dirbuster`, `Wfuzz`.

## 🎯 Чек-лист для тестирования СЗИ (Что проверять?)

*   **Для IDS/IPS движков:** Регистрирует ли система скрытые типы сканирования (XMAS, NULL)? Отличает ли полуоткрытое SYN-сканирование от легитимных попыток подключения?
*   **Для модулей контроля приложений (AppID):** Способен ли NGFW определить сигнатуру утилиты сканирования, если она маскирует свой User-Agent под обычный браузер (например, по структуре и порядку следования HTTP-заголовков)?
*   **Для поведенческого анализа (NTA/Пороги NGFW):** Срабатывает ли правило блокировки источника при превышении лимита на количество TCP-подключений в секунду к разным портам устройства?

## 📝 Спецификация именования PCAP-файлов

Все дампы в этой папке должны быть названы по шаблону:  
`T1595.[подтехника]_[инструмент]_[режим/ключи].pcap`

**Примеры:**
*   `T1595.001_nmap_ping_sweep.pcap`
*   `T1595.002_nmap_syn_stealth_scan.pcap`
*   `T1595.002_nikto_web_vulnerability_scan.pcap`
*   `T1595.003_gobuster_dir_bruteforce.pcap`
