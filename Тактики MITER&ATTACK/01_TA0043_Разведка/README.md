# Тактика: Разведка (Reconnaissance — TA0043)

**Разведка** — это начальный этап исследования целевой инфраструктуры. Нарушитель собирает информацию о системах, сетях и веб-ресурсах организации для планирования векторов атаки и подготовки инструментария. 

В данном разделе бенчмарка собраны дампы сетевого трафика (PCAP), фиксирующие активность атакующих утилит, сканеров и автоматизированных скриптов в процессе сбора данных о периметре.

---

## 📂 Структура каталога (Техники с сетевым следом)

Каждая подпапка содержит профили трафика для конкретных техник матрицы, которые могут быть зафиксированы сетевыми средствами защиты:

*   **`01_T1595_Активное_сканирование` (Active Scanning)**
    *   *Описание:* Прямое автоматизированное прощупывание сетевых узлов и портов.
    *   *Состав дампов:* TCP SYN / FIN / XMAS сканирование (Nmap), работа скриптов автоматизации (Nmap NSE), трафик сканеров уязвимостей сетевого и веб-уровня (Nessus, Acunetix, Nikto), а также перебор директорий веб-серверов по словарям (Gobuster, Dirb, Wfuzz).
*   **`02_T1590_Сбор_информации_о_сети` (Gather Victim Network Information)**
    *   *Описание:* Попытки нарушителя определить внутреннюю топологию, маршрутизацию и структуру доменных имен.
    *   *Состав дампов:* Запросы на зонный перенос DNS (AXFR Zone Transfer), массовые/агрессивные запросы reverse DNS (PTR-записи) и трассировка маршрутов.
*   **`03_T1598_Фишинг_для_сбора_информации` (Phishing for Information)**
    *   *Описание:* Сетевое взаимодействие, направленное на выведывание учетных данных или технических деталей через социальную инженерию.
    *   *Состав дампов:* SMTP/IMAP-сессии доставки фишинговых писем-приманок, а также HTTP/HTTPS-трафик обращения к развернутым фишинговым страницам аутентификации.
*   **`04_T1592_Сбор_информации_о_хостах` (Gather Victim Host Information)**
    *   *Описание:* Определение операционных систем, типов оборудования и версий запущенных сервисов.
    *   *Состав дампов:* Сетевой перехват баннеров ответов от открытых портов (Banner Grabbing), специфичные запросы утилит для удаленного OS-fingerprinting.
*   **`05_T1594_Поиск_по_сайтам_самой_жертвы` (Search Victim-Owned Websites)**
    *   *Описание:* Агрессивный сбор информации с публичных веб-ресурсов организации.
    *   *Состав дампов:* Массовый парсинг (Web Scraping) и полное зеркалирование/выкачивание сайта автоматизированными краулерами (Wget, HTTrack) для последующего офлайн-анализа метаданных и скрытых файлов.

## 🎯 Объект тестирования (Что валидируем на СЗИ?)

Используя дампы из этого раздела, на стендах проверяются следующие механизмы защиты:

1.  **Движок сигнатурного анализа (IDS/IPS / NGFW):** Способность обнаружить классические и скрытые типы сканирования портов, а также паттерны запросов популярных хакерских утилит.
2.  **Поведенческие правила и корреляция (NTA / SIEM):** Проверка порогов срабатывания (Thresholds). Способность системы сгруппировать распределенные во времени обращения к разным портам/директориям с одного IP-адреса в единый инцидент безопасности и инициировать блокировку.
3.  **Защита веб-приложений (WAF):** Проверка качества распознавания специфичных заголовков (User-Agent), сигнатур сканеров веб-ПО и детектирование активности агрессивных роботов (Anti-Scraping / Bot Detection).

## 🛠 Требования к PCAP-файлам в данном разделе

1.  **Именование файлов:** `[ID_техники]_[утилита]_[режим/особенность].pcap`  
    *Пример:* `T1595_nmap_syn_scan.pcap`, `T1594_wget_site_mirroring.pcap`.
2.  **Чистота дампа:** Перед загрузкой удаляйте из PCAP фоновый "шум" (широковещательный трафик, сторонние сессии), оставляя только пакеты, генерируемые инструментом разведки.
3.  **Локальный README:** Внутри конечной папки техники (например, в `01_T1595_Активное_сканирование`) рекомендуется создать краткое описание с указанием ключей запуска утилиты и IP-адресов стенда.
