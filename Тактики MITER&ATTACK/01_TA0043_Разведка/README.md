# Тактика: Разведка (Reconnaissance — TA0043)

**Разведка** — это начальный этап исследования целевой инфраструктуры. Злоумышленник собирает информацию, которая будет использована для планирования дальнейших действий, выбора векторов атаки и подготовки инструментов. Как правило, этот этап выполняется до получения первоначального доступа к сети жертвы.

В рамках данного бенчмарка здесь собираются дампы трафика (PCAP), которые генерируют атакующие утилиты и сканеры в процессе прощупывания периметра.

---

## 📂 Структура подкаталогов (Техники)

Каждая подпапка содержит дампы сетевой активности для конкретных техник матрицы:

*   **`01_T1595_Активное_сканирование`** (Active Scanning)
    *   Дампы сканирования портов, хостов и поиска уязвимостей.
    *   *Примеры трафика:* Nmap (SYN/FIN/XMAS-сканирование), скрипты NSE, сетевая активность сканеров уязвимостей (Nessus, Acunetix, Nikto) и утилит перебора директорий (Gobuster, Dirb).
*   **`02_T1590_Сбор_информации_о_сети`** (Gather Victim Network Information)
    *   Сетевые маркеры попыток определить топологию сети и конфигурацию DNS.
    *   *Примеры трафика:* Запросы на зонный перенос DNS (AXFR Zone Transfer), массовые запросы reverse DNS (PTR).
*   **`03_T1598_Фишинг_для_сбора_информации`** (Phishing for Information)
    *   Дампы сетевого взаимодействия, направленного на сбор учетных данных или технических деталей через социальную инженерию.
    *   *Примеры трафика:* SMTP/IMAP сессии с доставкой фишинговых писем-приманок, HTTP-трафик взаимодействия пользователя с фишинговой формой авторизации.

---

## 🎯 Что проверяется с помощью этих дампов на СЗИ (NGFW / IDS / IPS / NTA)

1.  **Качество сигнатурного анализа:** Способность сигнатурного движка IDS/IPS обнаружить агрессивные или скрытые методы сканирования (например, TCP SYN/XMAS сканирование от Nmap).
2.  **Поведенческий анализ и корреляция (Пороги срабатывания):** Способность систем защиты группировать тысячи единичных обращений к разным портам с одного IP-адреса в единый инцидент "Сканирование портов" (Port Scanning) и автоматически блокировать источник.
3.  **Детектирование работы сканеров (WAF/DPI):** Проверка качества распознавания специфичных заголовков (User-Agent) и сигнатур запросов популярных автоматизированных сканеров веб-приложений.

*   Файлы должны быть очищены от лишнего "шума" (оставляйте только пакеты сканирования/разведки).
